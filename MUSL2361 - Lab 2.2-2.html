Lab 2.2 - Character Animation

In this lab, we will see how character animation can be implemented in Three.js.

Motion data are available in many different formats. Some formats, like BVH, purely stores the motion of a body skeleton. Some, oo the other hand, already baked the motion on a particular 3D object model (such as Collada DAE).

Biovision BVH

The BVH file format was originally developed by Biovision, a motion capture services company, as a way to provide motion capture data to their customers. The name BVH stands for Biovision hierarchical data.

There's a great repo on BVH data, maintained by CMU - http://mocap.cs.cmu.edu

Baked Motion in Collada Dae

Motion animation could be presented in BVH file. But most of time, we have 3D models with animation being baked to it. These 3D models have been rigged, meaning that there're probably bones or other structures being developed in these models, such that animation could be performed.

Let's visit Mixamo (https://www.mixamo.com), an online platform which can rig 3D model and apply animation on it. You may use the character provided there, or upload the Naruto character that has been included in the CG03 folder.

Download the character in DAE format and place it under models/dae. Under MacOS, we can hit space bar on the dae file to see the baked texture and animation of this model.

Collada Dae with Three.js

Three.js understand animated models in JSON, or Collada Dae formats. Most major 3D modelling software can export models in these formats.

Create a new file and name it as cg04.html. Use the following starter template.

<!DOCTYPE html>
<html>
  <head>
    <title>Running Naruto</title>
    <style>
      html, body { margin: 0; padding: 0; overflow: hidden; }
    </style>
  </head>

    <body>
        <script src="https://cdnjs.cloudflare.com/ajax/libs/three.js/95/three.min.js"></script>
        <script src="http://threejs.org/examples/js/loaders/ColladaLoader.js"></script>
        <script>
            var scene = new THREE.Scene();
            scene.background = new THREE.Color(0xeeeeee);

            var aspect = window.innerWidth / window.innerHeight;
            var camera = new THREE.PerspectiveCamera(75, aspect, 0.1, 1000);
            var renderer = new THREE.WebGLRenderer();
            renderer.setSize(window.innerWidth, window.innerHeight);
            document.body.appendChild(renderer.domElement);

            // MORE Javascript will go here.
            
            scene.add(new THREE.GridHelper(400, 10));

            camera.position.set(150, 100, 300);
            camera.lookAt(0, 0, 0);

            var render = function () {
                requestAnimationFrame(render);
                renderer.render(scene, camera);
            };

            render();
        </script>
    </body>

</html>

Notice that the script tag for the Collada Loader is added in line 12. In line 25, we add a helper grid for better visualization of the objects. In line 28, we use the camera.lookAt() function such that our camera is now facing the origin.

Next, assume the file is located and named as Run/Run.dae, use the following block to import this model.

var mixer;
var naruto = new THREE.Object3D();

var loader = new THREE.ColladaLoader();
loader.load('models/dae/Run/Run.dae', function (collada) {

    var animations = collada.animations;
    var naruto = collada.scene;

    naruto.scale.set(1, 1, 1);
    naruto.position.z = - 100;

    scene.add(naruto);

    mixer = new THREE.AnimationMixer(naruto);   // create global mixer
    mixer.clipAction(animations[0]).play();     // play first animation clip

});

A DAE file can carry a number of animations (i.e.animations[]) and a scene. This scene carries the naruto 3D model, together with its texture. In some cases, lighting can also be added to the collada scene.

This file only has one animation, thus in line 16, we play the first scene, which therefore is animations[0].

Now, the model is black as there's no light in the scene. Let's provide some lighting:

var ambientLight = new THREE.AmbientLight(0xffffff, 0.2);
scene.add(ambientLight);
    
var directionalLight = new THREE.DirectionalLight(0xffffff, 0.8);
directionalLight.position.set(1, 1, - 1);
scene.add(directionalLight);

Ambient light acts as the background lighting, where every spot on our scene will receive the same amount of light. On the other hand, directional light serves well to represent light source from a far distance, like our Sun.

Finally, to play the animiation, we have to advance the mixer. Firstly, we have to provide a clock to keep track of the time.

var clock = new THREE.Clock();

Put it above the render() function.

In the render() function, add the following in the middle:

var delta = clock.getDelta();

if (mixer) mixer.update(delta);

For each time the render() function is visited, clock.getDelta() will give us the amount of time passed after the last getDelta() function call. We then use the delta value to advance our mixer, thus the animation could be played.

Moving Around

Let's use the WASD keys to move Naruto around. Firstly, let's set up a (x, y, z) vector to keep track of the moving direction when we press or remove a key. Put the below code somewhere before the render() function call.

var moveVector = new THREE.Vector3();

Next, set up the following event-driven functions to modify the vector values.

document.addEventListener("keydown", keyDown, false);
document.addEventListener("keyup", keyUp, false);

function keyDown(e) {

    if (e.key == 'a') moveVector.x = 1;
    if (e.key == 'd') moveVector.x = -1;
    if (e.key == 's') moveVector.z = -1;
    if (e.key == 'w') moveVector.z = 1;

}

function keyUp(e) {

    if (e.key == 'a') moveVector.x = 0;
    if (e.key == 'd') moveVector.x = 0;
    if (e.key == 's') moveVector.z = 0;
    if (e.key == 'w') moveVector.z = 0;

}

Finally, modify our render loop such that Naruto will move according to the current moveVector.

naruto.position.add(moveVector);
