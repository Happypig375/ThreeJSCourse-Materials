Lab 5.1 - Shadow Clone Technique

We've learnt how to develop fragment shaders to provide some visual effects. Today, I'm going to show you the Shadow Clone Technique. The shadow clone technique isn't a skill in computer graphics, instead it is a symbolic jutsu (忍術) in the Naruto manga series.

To produce the above CG scene, you will have to develop a fragment shader which implements

  • Phong lighting model,
  • Toon shading, and
  • Texture mapping.

On top of it, you will also have to:

  • apply custom shaders to model that involves multiple meshes
  • construct a parametric plane
  • perform ray casting to pinpoint a location in the 3D scene, and
  • cast shadow.

Prepare the Custom Shaders

There are many ways to load custom shaders in Three.js, as being discussed in https://stackoverflow.com/questions/5878703/webgl-is-there-an-alternative-to-embedding-shaders-in-html and https://stackoverflow.com/questions/14219947/why-do-shaders-have-to-be-in-html-file-for-webgl-program.

One approach is to put the glsl shader code directly inside the same HTML document. This approach doesn't require the use of web server but the code is more messy and difficult to follow. In this course, you are advised to save the codes seperately and use FileLoader load the shader files.

Construct a pair of files - cg05_vertex.glsl and cg05_fragment.glsl.

cg05_vertex.glsl:

precision highp float;
// attribute vec3 position;
// attribute vec3 normal;
// uniform mat3 normalMatrix;
// uniform mat4 modelViewMatrix;
// uniform mat4 projectionMatrix;
varying vec3 fNormal;
varying vec3 fPosition;

void main()
{
  fNormal = normalize(normalMatrix * normal);
  vec4 pos = modelViewMatrix * vec4(position, 1.0);
  fPosition = pos.xyz;
  gl_Position = projectionMatrix * pos;
}

Line 2 - 8 declare the input and output values of this vertex shader.

  • attribute are the input values, per vertex. Thus, given a 3D cube model, there should be 8 vertices, meaning 8 different positions, together with 8 different corresponding normals.

  • vec3 represents a vector with 3 components. We also have vector types vec4 and vec2.

  • uniform represents a single input value, expects to be used by all vertices of the incoming model.

  • mat3 and mat4 defines the types for 3 x 3 and 4 x 4 matrices respectiviely.

  • varying is the output of the vertex shader.

cg05_fragment.glsl:

precision highp float;
uniform float time;
uniform vec2 resolution;
varying vec3 fPosition;
varying vec3 fNormal;

void main()
{
  gl_FragColor = vec4(fNormal, 1.0);
}

Here, we simply start with the default shaders from the Shdr online editor. Some inputs are commented as they have already been defined under Three.js.

Using the Custom Shaders in Three.js

Simply use the built-in FileLoader to load the custom shaders and then reference them with variables vShader and fShader. Here's the required code segment, put it in our main program.

var vShader, fShader;
var shaderLoader = new THREE.FileLoader();

shaderLoader.load('cg05_vertex.glsl', function (data) {
    vShader = data;
});

shaderLoader.load('cg05_fragment.glsl', function (data) {
    fShader = data;
});

So far, we've been using the MeshLambertMaterial and MeshPhongMaterial to give object/mesh realistic visual behaviour. To use our own shaders, we will use ShaderMaterial instead. 

object.traverse(function (child) {
    if (child instanceof THREE.Mesh) {

        child.material = new THREE.ShaderMaterial({
            vertexShader: vShader,
            fragmentShader: fShader
        });

    }
});

Now, reload your page and Naruto should be mainly in blue (Normals pointing outwards). This implies we loaded the shaders successfully.

Our Naruto 3D model is complicated and involves 8 meshes. You can always console.log to display the mesh array. The above code segment basically assigns each mesh the same pair of shaders.

Texture Mappng with Shaders.

We'd like to show the original texture on our Naruto model. This could be done by performing texture mapping in the fragment shader.

In our main program, replace the previous code segment with the following:

var textureLoader = new THREE.TextureLoader();
textureLoader.setPath('obj/86b61s2onq-Naruto/');

var materialMap = {};
object.traverse(function (child) {
    if (child instanceof THREE.Mesh) {
        if (child.material.id in materialMap) {
            console.log("Using predefined material");
            child.material = materialMap[child.material.id];
        }
        else {
            console.log("Creating new material");
            console.log(child.material);

            var texture;

            if (child.material.name != "Material__3")
                texture = textureLoader.load("ntxr001.png");
            else
                texture = textureLoader.load("ntxr004.png");

            materialMap[child.material.id] = new THREE.ShaderMaterial({
                uniforms: {
                    lightSrc: { type: "v3", value: directionalLight.position },
                    texture: { type: "t", value: texture }
                },
                vertexShader: vShader,
                fragmentShader: fShader
            });
            child.material = materialMap[child.material.id];
        }

    }
});

Among the 8 meshes in the model, there are only 3 different materials needed to be created. The 3 materials are for the Naruto body, eyes and the tongues of his headband. These 3 materials are being created in the else part of the above program segment. Currently, the 3 materials basically use different texture files: ntxr001.png for the body and ntxr005.png for the eyes. Line 17 - 20 selects the correct texture for these materials.

The textures are passed to the shaders as a uniform attribute in line 23 - 26. In fact, the position of light source is also provided here. This is needed to perform Phong lighting model later.

We can add as many uniforms as we like, simply by expanding line 23 - 26. Type t corresponds to the sampler2D type in the shaders. More about the ShaderMaterial could be found here: http://threejs.org/docs/#Reference/Materials/ShaderMaterial

For each object, we can assign a pair of uv values to each vertex. Each value is between 0 and 1. These values will be used in texture mapping. In the vertex shader, we simply have to pass the uv values to the fragment shaders. Thus, modify the vertex shader as follows

precision highp float;
// attribute vec3 position;
// attribute vec3 normal;
// uniform mat3 normalMatrix;
// uniform mat4 modelViewMatrix;
// uniform mat4 projectionMatrix;
varying vec3 fNormal;
varying vec3 fPosition;
varying vec2 vUv;

void main()
{
  fNormal = normalize(normalMatrix * normal);
  vec4 pos = modelViewMatrix * vec4(position, 1.0);
  fPosition = pos.xyz;
  vUv = uv;
  gl_Position = projectionMatrix * pos;
}

Here, we can a new varying output vUv, which is simply the incoming uv values (check line 16).

In the fragment shader, modify it as follows.

precision highp float;
uniform float time;
uniform vec2 resolution;
varying vec3 fPosition;
varying vec3 fNormal;
varying vec2 vUv;

uniform sampler2D texture;

void main()
{
  gl_FragColor = texture2D(texture, vUv);
  // gl_FragColor = vec4(fNormal, 1.0);
}

From line 8, we obtain the texture from the main program. In line 12, the texture2D() function would perform the actual texture mapping, which reads a texel from the texture according to the current vUv value.

The Terrain

We'll start with a basic flat floor. To create a plane geometry:

var geometry = new THREE.PlaneGeometry(2000, 2000, 32);
var material = new THREE.MeshLambertMaterial({color: 0x6B5A38});

var plane = new THREE.Mesh(geometry, material);
plane.position.set(0, -100, 0);
plane.rotation.set(-Math.PI / 2, 0, 0);

scene.add(plane);

The first two parameters of the PlaneGeometry() function define the width and height of the plane. By default, the plane is vertical and facing the +z direction. Line 6 above rotates it by 90 degree along the x-axis.

Desert

A lot of times, Naruto has to fight in some rural areas, where the floor is not that flat. In this case, we have to replace the PlaneGeometry() with ParametricGeometry().

ParametricGeometry(func, slices, stacks)

func — A function that takes in a u and v value each between 0 and 1 and returns a Vector3

slices — The count of slices to use for the parametric function

stacks — The count of stacks to use for the parametric function

Reference: http://threejs.org/docs/#Reference/Extras.Geometries/ParametricGeometry

The last two parameters simply define the complexity of this geometry and could be ignored for now. The first one does deserve more explanation.

Still remember the uv values we discussed in last section? It's an extra attribute assigned to the mesh vertices. Here, we are going to change the vertex positions based on these uv values.

Time to show off my Maths senses:

function func(u, v) {

    return new THREE.Vector3((u-0.5)*2000, (v-0.5)*2000, Math.cos(10*u) * Math.cos(10*v) * 100);

}

Most work is being done in the z-component. Remember the plane is vertical by default, and thus the variations have to be developed on the z-compoenet.

Finally, replace the original PlaneGeometry() with this line.

var geometry = new THREE.ParametricGeometry( func, 32, 32 );

Kage Bunshin no Jutsu

A clone of Naruto will be generated and placed on the terrain whenever we click on it. Thus, the first thing is to set up the click listener and the corresponding callback function.

var mouse = new THREE.Vector2();

function onDocumentMouseClick(event) {

    event.preventDefault();

    mouse.x = (event.clientX / window.innerWidth) * 2 - 1;
    mouse.y = - (event.clientY / window.innerHeight) * 2 + 1;

    raycaster = new THREE.Raycaster();

    var vector = new THREE.Vector3(mouse.x, mouse.y, 1).unproject(camera);

    raycaster.set(camera.position, vector.sub(camera.position).normalize());

    var intersects = raycaster.intersectObjects(scene.children);

}

document.addEventListener('click', onDocumentMouseClick, false);

Nothing new as we have tried it before. Each time when click our screen, the onDocumentMouseClick() function will be invoked. This function basically converts the mouse position from screen coordinates (+y downwards) to the THREE.js view coordinates (+y upwards). A virtual ray, which is perpendicular to the screen, is also created from the mouse position and goes towards the far plane. This ray may intersect with a number of objects in our scene and those objects are captured in the intersects array.

We'd like to place a new Naruto at the intersect point. Thus, append the below to the onDocumentMouseClick() function.

if (intersects.length > 0) {

    var narutoClone = naruto.clone();

    narutoClone.position.x = intersects[0].point.x;
    narutoClone.position.y = intersects[0].point.y - 10;
    narutoClone.position.z = intersects[0].point.z;

    narutoClone.rotation.z = Math.random() - 0.5;

    scene.add(narutoClone);
}

Shadow Casting

The scene won't look realistic if shadow is not rendered. To achieve it, we first have to enable shadow casting with 

renderer.shadowMap.enabled = true;

Next, we have to specify which meshes to cast and receive shadows. So for all child meshes of the Naruto model, we need the following line:

child.castShadow = true;

The terrain will receive the shadow, thus

plane.receiveShadow = true;

Finally, we have to specify which light source is casting the shadow:

directionalLight.castShadow = true;

directionalLight.shadow.camera.near = 0.5;       // default
directionalLight.shadow.camera.far = 500;      
directionalLight.shadow.camera.left = -512;    
directionalLight.shadow.camera.right = 512;    
directionalLight.shadow.camera.top = -512;     
directionalLight.shadow.camera.bottom = 512;

The last six lines specify a volume where shadow will be calculated and rendered. Objects outside this volume will not cast or receive shadow.

It's hard to estimate appropriate values for the shadow camera. The following should give us a big hand.

var helper = new THREE.CameraHelper(directionalLight.shadow.camera);
scene.add(helper);
