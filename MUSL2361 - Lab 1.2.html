Next, add the following code segment inside the // JavaScript will go here. section.

var scene = new THREE.Scene();
scene.background = new THREE.Color( 0x000000 );

var aspect = window.innerWidth / window.innerHeight;
var camera = new THREE.PerspectiveCamera( 75, aspect, 0.1, 1000 );
var renderer = new THREE.WebGLRenderer();
renderer.setSize( window.innerWidth, window.innerHeight );
document.body.appendChild( renderer.domElement );

var geometry = new THREE.SphereGeometry( 100, 32, 32 );
var material = new THREE.MeshNormalMaterial();
var sun = new THREE.Mesh( geometry, material );
scene.add( sun );
camera.position.z = 300;

var render = function () {
  requestAnimationFrame( render );
  renderer.render( scene, camera );
};

render();

Open this file with a browser, and you will see the sun sitting at the middle of the screen. This is achieved by using the THREE.SphereGeometry() function, which expects three input arguments. The first one defines the radius of the sphere. The others define the number of horizontal and vertical segments.

Line 11 set the material of of sun. Currently the surface is painted according to the normal of each pixel. For example, the top pixel is pointing upward, thus having a vector of (0, 1, 0). This vector is also used as the color (rgb). Thus, the top of the sphere is painted in pure green.

To clearly see the segments, we can add wireframe to the object. Modify line 11 as follows:

var material = new THREE.MeshNormalMaterial({wireframe: true});

Light and Material

Applying Material

Now, let's apply a yellow Lambert (not shinny) material to our moon.

Locate the line which defines the material for the moon. Comment out this line and use the following instead

var material = new THREE.MeshLambertMaterial({
  color : 0xEBC815
});

Now, the moon should have been colored, but in fact, the moon cannot be seen. The reason is, MeshLambertMaterial is designed to reflect light; Unlike MeshNormalMaterial, which basically paints solely on the plane's normal (which direction it faces). Thus, the next step is to add lights into our scene.

Adding Lights

We are going to add a point light at the center of the Sun. This light will shine on the Earth and the Moon.

var pointLight = new THREE.PointLight(0xffffff);

// set its position
pointLight.position.x = 0;
pointLight.position.y = 0;
pointLight.position.z = 0;

// add to the scene
scene.add(pointLight);

Now, you should be able to see the moon, in yellow.

Phong Material

We are going to apply another material to our Earth. This material best models shinny objects like metals and plastic. Again, locate the existing material for our Earth and comment them out.

Use the following instead.

var material = new THREE.MeshPhongMaterial({
    color : 0x6b93d6,
    shininess: 30
});

shininess defines the specular highlight; a higher value gives a sharper highlight. You may try different values and observe the difference.

Animation

Self-rotation

We learnt how to perform rotate an object locally last lab, as we had to rotate the hand to fit the head. Thus, let's simply apply similar settings to perform self-rotation on the earth and the moon. The syntax is simply as follows

earth.rotation.y += 1;
moon.rotation.y += 1;

Revolution

The main question is how to make our moon spin around the earth. In fact, the moon doesn't spin around the earth. Physics tells us that both the earth and the moon orbit around the center of masses of the Earth-Moon system. Thus, in order of simulate this behavior, we are going to use the Group concept under three.js

Comment our all the statements related to the positions of the earth and the moon. We can also remove them from the scene by commenting our the scene.add() statements.

Next, add the following after both earth and moon have been created.

var pivot = new THREE.Group();
pivot.add(earth);
pivot.add(moon);

scene.add(pivot);
pivot.position.x = 200;

moon.position.x = 50;
earth.position.x = -50;

The first three lines create a new group named as pivot. Both earth and moon are added to this group. Line 5 displays this group on our scene. Line 6 places the entire group at x=200.

Line 8 and 9 re-position our two objects with respected to the pivot position. As the center of masses lie between the two objects, their x values are of opposite signs.

Now, to perform the required animation, let's simply rotate the pivot group locally!!!
